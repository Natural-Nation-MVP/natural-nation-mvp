# Natural Nation MVP Frontend Foundation\n\nThis directory (`app/frontend/`) contains the foundational static web files for the Natural Nation mobile-first holistic wellness platform's MVP. This initial implementation slice focuses on presenting a testable member-facing "Day 1 Dashboard" preview.\n\n## Files Included:\n\n- `index.html`: The main HTML structure for the Day 1 Dashboard, featuring:\n    - A welcoming header.\n    - Displays for "Wellness Score" and "Rejuvenation Score".\n    - An introduction to "Duey, your AI wellness mentor".\n    - A "Daily Protocol" timeline with placeholder activities.\n    - "Progress Check-in" controls.\n    - Designed with non-medical language and responsive layouts for various screen sizes.\n- `styles.css`: Provides the styling for `index.html`, implementing a mobile-first approach and media queries for tablet and desktop responsiveness.\n- `app.js`: A minimal JavaScript file for basic interactions and console logging, confirming the script's loading and providing a placeholder for future dynamic features.\n- `README.md`: This file, describing the purpose and contents of the frontend foundation.\n- `tests/validate-foundation.mjs`: A Node.js validation script to ensure the presence of required experience elements within these foundational files.\n\n## Day 1 Dashboard Preview Features:\n\n- **Responsive Design**: Adapts seamlessly across mobile, tablet, and desktop devices.\n- **AI Wellness Mentor (Duey)**: Clearly identifies Duey as a non-medical AI wellness mentor.\n- **Wellness and Rejuvenation Scores**: Visual representation of user scores.\n- **Daily Protocol Timeline**: A structured list of activities for the day.\n- **Progress Check-in Controls**: Interactive elements for users to log their daily feelings and thoughts.\n- **Non-Medical Language**: All content is phrased to be supportive and wellness-focused, avoiding clinical or medical terminology.\n\n## How to View:\n\nSimply open `app/frontend/index.html` in your web browser. Resize your browser window or use developer tools to test its responsiveness across different device views.\n\n## Validation:\n\nThe `tests/validate-foundation.mjs` script can be run using Node.js to verify that this foundation meets the specified requirements.
