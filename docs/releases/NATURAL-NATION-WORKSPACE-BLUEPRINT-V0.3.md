# Natural Nation Workspace Blueprint v0.3

Status: Implemented for Founder Review

## Revision Summary

### Shared Execution Action Bar

The Blueprint no longer uses a custom sticky card inside the page.

It now uses the same `.bottom-bar` component, application-shell layer, desktop positioning, tablet behavior, mobile behavior, and four-column structure as Build Studio.

Blueprint action-bar content:

- Blueprint Readiness
- Approval Effect
- Pending Decisions
- Approve Blueprint

Only the execution bar belonging to the active page is visible.

### Platform Standard

NNOS-UX-002 establishes the Build Studio bottom action bar as the required execution pattern for all current and future Founder OS pages with page-level execution actions.

### Our Mission Card

Our Mission now uses the normal Founder OS card hierarchy:

1. eyebrow tag
2. section title
3. concise supporting text

The oversized mission-statement treatment was removed.

### Compact Cards

Blueprint cards were tightened by reducing:

- internal padding
- gaps
- icon sizes
- chart size
- line spacing
- phase-card height
- component-card spacing

This compact treatment remains the default until a later styling phase is approved.

## Safety Boundary

The Blueprint action remains review-only. Final approval cannot be recorded until the MVP billing decision is resolved and protected execution controls are available.
