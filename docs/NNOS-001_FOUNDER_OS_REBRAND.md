# NNOS-001 — Founder OS Rebrand and Terminology Migration

**Status:** Approved  
**Scope:** Founder/Developer platform naming, UI labels, and operating terminology  
**Compatibility Rule:** Keep all existing IDs and repository paths unless a separate migration is approved.

---

## Approved Names

| Layer | Approved Name | Purpose |
|---|---|---|
| Canonical operating layer | Natural Nation Operating System (NNOS) | Defines project truth, decisions, standards, and operating records. |
| Founder interface | Natural Nation Founder OS | The Founder-facing executive operating environment. |
| Intelligence workspace | Project Intelligence | Search, relationships, decisions, records, and traceability. |
| Consumer product | Natural Nation App | Customer-facing wellness application. |

---

## Naming Rules

1. Keep existing IDs such as `NNCC-*`, `NN-KS-*`, and `NNOS-*`.
2. Keep the physical `knowledge/` folder until a separate repository migration is approved.
3. Use user-facing action titles in navigation and major page headers.
4. Treat “Mission Control” as a visual design language, not the product name.
5. Use “Founder OS” as the product identity for the Founder/Developer environment.
6. Use “NNOS” for the canonical operating system that powers the Founder OS.
