# Founder OS Phase Completion Standard

Status: Approved

## Purpose

This standard governs how Founder OS development proceeds after Release 3.

## Core Rule

Founder OS advances one phase at a time.

No later phase begins until the active phase is fully implemented, reviewed, validated, corrected, synchronized, and locked.

## Phase Lifecycle

Each phase follows this sequence:

1. Architecture
2. Implementation
3. Internal QA
4. Founder Review
5. Founder Validation
6. Corrections
7. Phase Closeout
8. Phase Lock

## Founder Validation Results

Founder validation may result in:

- PASS
- PASS WITH FINDINGS
- FAIL

## Completion Criteria

A phase is not complete until:

- Every approved subphase is implemented.
- Every approved subphase is usable inside Founder OS.
- Repository and Knowledge Base records are synchronized.
- Founder Review is completed.
- Founder Validation passes.
- All findings are resolved.
- Decision Ledger is updated.
- Validation Center is updated.
- Project State is updated.
- Session Log is updated.
- Roadmap is updated.
- Phase is marked COMPLETE and LOCKED.

## No Parallel Phase Rule

Do not build Phase 5, Phase 6, Phase 7, or Phase 8 work while Phase 4 is still active unless the Founder explicitly approves an exception.
