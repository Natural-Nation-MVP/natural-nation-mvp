# Continuous Synchronization Standard

Status: Founder Approved

## Rule

No approved work is complete until the project files, knowledge base, roadmap, and session tracking are synchronized.

## Required Flow

1. Plan
2. Build
3. Validate
4. Founder approval when required
5. Update repository files
6. Update knowledge base records
7. Update roadmap
8. Update session log
9. Add decision or governance record when needed
10. Mark complete

## Records To Update When Affected

- PROJECT_STATE.md
- SESSION-LOG.md
- release roadmap
- governance records
- decision records
- knowledge base files
- Founder OS records
- build package records
- design records
- product records
- API records
- testing records

## Definition of Done

A task is done only when the work and the records that describe the work are both current.

## AI Team Responsibilities

- Art: architecture and decision records
- Codex: implementation and technical notes
- GPose: founder-facing summaries and prompts
- Gemini: design and UX records
- Duey: wellness knowledge and protocol guidance
- Founder: final approval for locked decisions

## Founder OS Goal

Founder OS should track missing sync work, unsynchronized approvals, documentation health, roadmap state, and AI context readiness.

## Operating Principle

GitHub is the source of truth. ChatGPT helps create and refine the work. Founder OS keeps the project synchronized.
