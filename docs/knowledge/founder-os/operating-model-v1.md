# KB-FOUNDER-005 — Founder OS Operating Model v1

Status: Founder Approved

## Purpose

This document defines the approved operating model for Founder OS. It describes how work moves through the system and how each workspace contributes to the Natural Nation development lifecycle.

## Operating Loop

Mission Control
↓
Knowledge Graph
↓
Repository Intelligence
↓
Build Studio
↓
AI Operations
↓
GitHub and Knowledge Base
↓
Mission Control

## Workspace Responsibilities

| Workspace | Primary Founder Question | Success Outcome |
| --- | --- | --- |
| Mission Control | What needs my attention right now? | Clear executive priorities |
| Knowledge Graph | What do we know? | Trusted, searchable knowledge |
| Repository Intelligence | What changed and what is out of sync? | Repository health and synchronization |
| Build Studio | What are we building next? | Approved implementation packages |
| AI Operations | Who is working on what? | Coordinated AI execution |
| GitHub and Knowledge Base | What is the official record? | Canonical project history |

## Feature Placement Rule

Every new Founder OS feature must answer three questions before it is added:

1. Which workspace does it belong to?
2. What Founder question does it answer?
3. What decision or action does it enable?

If a feature cannot answer those questions, it should not be added until its purpose is clarified.

## Design Principle

Founder OS should not be a collection of disconnected screens. It should operate as a decision and execution loop for building Natural Nation.
