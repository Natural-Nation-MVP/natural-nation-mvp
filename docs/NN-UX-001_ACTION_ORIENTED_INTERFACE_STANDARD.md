# NN-UX-001 — Action-Oriented Interface Standard

**Status:** Approved  
**Applies To:** Natural Nation Founder OS and future Founder/Developer interfaces

---

## Purpose

Founder OS navigation should describe the action the Founder is taking, not merely the object being opened.

This makes the interface feel like an operating environment instead of a static dashboard.

---

## Core Rule

Primary navigation items, page titles, workflow labels, and future command palette entries should begin with an action verb whenever practical.

---

## Approved Verb Patterns

| Action | Use When |
|---|---|
| Monitor | Checking live or current status. |
| Review | Reading summaries, history, decisions, or milestones. |
| Approve | Making Founder-level decisions. |
| Explore | Searching or browsing connected intelligence. |
| Coordinate | Managing AI team roles or handoffs. |
| Generate | Creating prompt packages, build packages, or outputs. |
| Manage | Running operations, releases, QA, or roadmaps. |
| Verify | Confirming implementation readiness. |

---

## Examples

| Passive Label | Action-Oriented Label |
|---|---|
| Dashboard | Monitor Project |
| Founder Brief | Review Daily Brief |
| Knowledge Browser | Explore Project Intelligence |
| Decision Registry | Review Decisions |
| AI Build Queue | Generate Build Packages |
| Verification | Verify Implementation |

---

## AI Alignment Benefit

Action-oriented labels double as AI commands.

Examples:

```text
Monitor Project
Review Daily Brief
Explore Project Intelligence
Generate Build Packages
Verify Implementation
```

These commands can later power a Founder OS command palette and AI workflow launcher.

---

## Compatibility Rule

Action-oriented labels do not require route, ID, or folder renames.

Routes such as `build-queue`, `knowledge`, and `verification` may remain stable while the user-facing label changes.
